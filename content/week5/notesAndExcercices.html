Week 5 Notes

When we write code in a development environment, we should write it in a way that when it fails, we can quickly identify that there is a bug. (Design the program so that if it fails, it fails loudly). In a production environment, issues should fail gracefully.

Strict Mode

            
    'use strict';

We can put this link of code at the top of our JavaScript file so when the code is running, it will through errors for poorly styled code. This will increase errors but will help us ensure that the code is well written and thus avoids big hidden bugs.

Linting Tools

Are pronging’s that can be installed and help check that the code is following certain coding conventions.

Exception Handling

Try

We can use try to wrap code that might through an exception and that we do not want that exception to crash the application.

Catch

“catch” is the “else” statement of “try”, if we think of “try” as the “if” statement. When we use “try” we are saying, if this code works then continue the program, else throw an exception. The exception that is thrown is caught by a catch statement.

Test

A test is a function that checks if a piece of code is working in the way it should.

Test-driven Development

Is the process of writing test before writing the actual code. The life cycle of test-driven development is:

  1. Write tests (that initially fail)
  2. Write code to pass the tests
  3. Refactor the code
  4. Test refactored code
  5. Write more tests for new features

Testing Frameworks

Jest is a popular testing framework developed by Facebook. It can be installed in in the computer using npm
